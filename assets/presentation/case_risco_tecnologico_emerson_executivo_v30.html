Slide 05 • Pipeline

Esteira implementada em Glue: quatro jobs com responsabilidades claras

Cada job resolve uma etapa: ingestão, visão executiva, visão operacional e entrega final do case.

job_01_case_riscos_bronze_to_silver_catalogacao
  └─ Lê XLSX/CSV, padroniza colunas, grava Silver em Parquet e cataloga.

job_02_case_riscos_silver_to_gold_modelagem
  └─ Cria gold_risco_tecnologico_por_sigla para priorização executiva.

job_03_case_riscos_gold_obsolescencia_por_servidor
  └─ Cria gold_obsolescencia_por_servidor com granularidade operacional.

job_04_case_riscos_gold_outputs_finais
  └─ Cria gold_hub_analitico_por_servidor e gold_controle_inconformidades_por_servidor.

Slide 06 • Modelo de dados

Modelo de dados: como o risco chega até o servidor

A lógica do modelo é simples: localizar o servidor, identificar o software instalado, conectar ao catálogo/versionamento e herdar os atributos de risco e organização.

1. Onde atuar?
servidores_siglas
Inventário de servidores com sigla de origem, arquitetura, ambiente e status operacional.
Chave: nome do servidor
→
2. O que está instalado?
cmdb_software_instance_sot
Lista os softwares instalados em cada servidor.
Join: servidores_siglas.nome = installed_on
→
3. Qual o produto e a versão?
cmdb_ci_spkg_sot
Catálogo tecnológico com nome, versão e identificador do produto.
Join: software instalado = name + version
Enriquecimento de risco
resultado_query3
Traz criticidade, índice de obsolescência, rating, impacto e cloud para o produto tecnológico/versionamento.
Resultado: risco por software/versão
Enriquecimento organizacional
resultado_query1 + resultado_query2
Preenche Diretoria, Comunidade, Tier/Criticidade da Sigla, Pillar, Domínio e Subdomínio no template final.
Resultado: contexto de negócio
Saída principal: gold_obsolescencia_por_servidorUma linha por servidor + software + versão + risco, pronta para gerar o HUB Analítico e o farol de inconformidade.
